Quiz and Question Microservices

The Quiz and Question Microservices project is a scalable and modular application designed using the microservices architecture. It leverages PostgreSQL for data storage and integrates multiple services using OpenFeign and Eureka Server for service discovery and communication.

Key Features:

  • Service-Oriented Architecture: Each feature is developed as an independent microservice.
  • Quiz Management: Create, update, and manage quizzes through dedicated services.
  • Question Handling: Add and manage questions with different types and categories.
  • User Interaction: Track user attempts, scores, and progress across multiple quizzes.
  • Reporting and Analytics: Generate reports on quiz performance and user engagement.
  • Seamless Communication: Microservices interact seamlessly using OpenFeign and are registered with Eureka Server for dynamic service discovery.

Technologies Used:

  • Spring Boot: For developing each microservice efficiently.
  • PostgreSQL: As the primary database for storing quiz and question data.
  • OpenFeign: To simplify HTTP communication between microservices.
  • Eureka Server: For service registration and discovery, ensuring high availability.

Supplier API

The Supplier API is a Spring Boot application designed for managing supplier data. It provides a comprehensive set of APIs for CRUD operations on suppliers, secured with HTTP Basic Authentication, and includes Swagger integration for documentation and testing.

Key Features:

  • CRUD Operations: Perform Create, Read, Update, and Delete operations on supplier data.
  • Security: Protected with HTTP Basic Authentication and BCrypt password encoding for secure access.
  • Swagger Integration: Provides interactive API documentation and testing at Swagger UI.
  • Data Access: Utilizes Spring Data JPA for efficient database interactions with PostgreSQL.

Key Components:

  • **Entities:** Defines the Supplier data model.
  • **Controller:** Manages API requests and responses.
  • **Service:** Contains business logic and service layer operations.
  • **Repository:** Handles data access and persistence using Spring Data JPA.

Setup Instructions:

  1. Clone the repository from GitHub: GitHub Repository.
  2. Set up a PostgreSQL database and update `application.properties` with your database credentials.
  3. Run the Spring Boot application.
  4. Access Swagger UI at http://localhost:8080/swagger-ui.html to interact with the APIs.
